# Markdown 文档目录生成规范

## 触发条件

**此技能在用户提到以下任何关键词时自动生效**：
- "MD文档" / "md文档"
- "Markdown" / "markdown"
- "MD格式" / "md格式"
- ".md文件" / ".md"
- "生成文档"（如果上下文涉及markdown）
- "创建文档"（如果是.md文件）
- 或任何涉及Markdown文档的操作

**触发条件已放宽**：只要提到MD/Markdown相关内容，就自动应用此规范。

---

## 文件命名规范

**所有 Markdown 文档必须遵循统一命名格式**：

```
序号-名称-类型-日期.md
```

### 命名规则

- **序号**：两位数字（01, 02, 03...），按创建时间由远及近排序
- **名称**：文档主题或项目名称（使用短横线连接多个词）
- **类型**：文档类型（见下方类型列表）
- **日期**：YYYYMMDD 格式（8位数字）

### 文档类型代码（中文）

- `总结` - 总结类
- `经验` - 经验类
- `笔记` - 笔记类
- `报告` - 报告类
- `教程` - 教程类
- `文档` - 知识库/文档类
- `会议` - 会议记录
- `分析` - 分析类

### 命名示例

```
01-Python学习-总结-20260304.md
02-Three.js最佳实践-经验-20260304.md
03-产品需求评审-会议-20260304.md
04-2025网页设计趋势-报告-20260304.md
05-React入门指南-教程-20260304.md
06-API接口文档-文档-20260304.md
07-用户行为数据-分析-20260304.md
08-前端性能优化-笔记-20260304.md
```

### 执行要求

1. 生成任何 Markdown 文档时，自动按此格式命名文件
2. 序号从已有文档的最大序号继续递增
3. 日期使用文档创建日期（当前日期）
4. 名称部分保持简洁，避免过长
5. 类型必须使用中文，从预定义列表中选择

---

## 适用范围

**仅在生成以下类型的 Markdown 文档时自动添加目录**：

- 总结类文档（工作总结、学习总结、项目总结）
- 经验类文档（技术经验、最佳实践、踩坑记录）
- 笔记类文档（读书笔记、会议记录、学习笔记）
- 报告类文档（研究报告、分析报告、调研文档）
- 教程类文档（技术教程、指南、文档说明）
- 知识库类文档（技术文档、百科、参考手册）

**不适用的场景**：

- 生成其他格式的文档（docx、pdf、pptx、xlsx 等） → 不添加目录
- 简短的 Markdown 文件（少于 3 个二级标题） → 不添加目录
- 代码示例、配置文件、README 等简单文档 → 不添加目录
- 日志、临时记录等非正式文档 → 不添加目录

---

## 目录格式规范

### 格式选择规则

**根据文档用途选择目录格式**：

1. **普通文档** - 使用 Obsidian 简化格式：`[[#标题文本]]`
   - 适用于：所有常规 MD 文档
   - 优点：简洁，Obsidian 中完美支持

2. **README 和 GitHub 文档** - 使用标准 Markdown 格式：`[标题文本](#标题文本)`
   - 适用于：README.md、要推送到 GitHub 的文档
   - 优点：GitHub 和大部分编辑器通用兼容

### 标题级别对应

- `##` 二级标题 → 目录中无缩进
- `###` 三级标题 → 目录中缩进 2 空格
- `####` 四级标题 → 目录中缩进 4 空格
- `#####` 五级标题 → 目录中缩进 6 空格

**注意**：一级标题 `#` 通常是文档标题，不放入目录。

## 示例

### 文档内容
```markdown
# Python 学习总结

## 🌤 基础语法
### 变量与数据类型
### 控制流程

## 📝 高级特性
### 装饰器
### 生成器

## 🎯 实战经验
```

### 生成的目录（Obsidian 格式）
```markdown
# Python 学习总结

## 目录

- [[#🌤 基础语法]]
  - [[#变量与数据类型]]
  - [[#控制流程]]
- [[#📝 高级特性]]
  - [[#装饰器]]
  - [[#生成器]]
- [[#🎯 实战经验]]

---

## 🌤 基础语法
...
```

### 生成的目录（GitHub 格式 - 仅用于 README）
```markdown
# Python 学习总结

## 目录

- [🌤 基础语法](#-基础语法)
  - [变量与数据类型](#变量与数据类型)
  - [控制流程](#控制流程)
- [📝 高级特性](#-高级特性)
  - [装饰器](#装饰器)
  - [生成器](#生成器)
- [🎯 实战经验](#-实战经验)

---

## 🌤 基础语法
...
```

## 执行规则

1. **判断是否需要目录**：检查文档类型和长度，符合适用范围才添加
2. **判断文档类型**：
   - 如果是 README.md 或明确要推送到 GitHub → 使用标准 Markdown 格式
   - 否则 → 使用 Obsidian 简化格式
3. **插入位置**：在文档标题（一级标题 `#`）之后，正文内容之前
4. **扫描标题**：提取所有 `##` 到 `#####` 标题
5. **生成链接**：
   - **Obsidian 格式**：`[[#完整标题文本]]`
   - **GitHub 格式**：`[标题文本](#锚点)` (锚点需转换：小写、空格改-、移除emoji)
6. **目录标题**：使用 `## 目录`
7. **保留原样**：标题中的 emoji 和特殊字符完整保留（Obsidian 格式）
8. **分隔线**：目录后添加 `---` 分隔线，区分目录和正文
